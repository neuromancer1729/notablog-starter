Beginner’s Guide To Understanding Blockchain and it’s Usecases

Posted on Mon, Nov 26, 2018 blockchain

What is blockchain?

With blockchain technology entering the mainstream, you may have heard of the terms “Bitcoin” and “Blockchain” in the news and wondered what they are, how they work, and why they matter.

This article will help answer some of these questions without going too much deep in the tech stuff and scary maths.

Let’s begin

In simple words, a blockchain is a append only linked list with shared state.

A linked list is a linear data structure where each element is a separate object. Each element (or node) is comprising of two items — the data and a reference to the next node.Shared state means that the data in the linked list is shared with other participants (nodes) of the blockchain.

The blockchain maintains a growing list of records, each new record added is linked to a previous record, forming a chain of records, these records are organized into blocks, hence the name blockchain.

How blocks are connected in a blockchain. Image credits : Yevgeniy Brikman

This growing list of records is maintained over several machines or nodes, each of these nodes warehouses the data and validates new data coming into the blockchain, thus making it more secure than traditional databases which rely on a single machine to do record keeping.

Each node in the blockchain is connected to each other forming a peer-to-peer network responsible for securing the data and overview addition of new blocks.

The records in blockchain can contain information such as transaction records, account balance, list of people authorized to access the account, document signatures etc.

Key features of Blockchain

Shared Ledger

Append-only distributed system on records shared across a business network. The more nodes there are in the blockchain the more secure this shared ledger will be.

Security

Blockchains ensure appropriate visibility of data to various participants, blockchain transactions are secure, authenticated and verifiable. Public key cryptography is used to ensure blockchain data access to participants.

Smart Contract

Blockchain allows for Business login to be translated to a code and embedded in the transaction database and be invoked by transactions.

Consensus

All nodes of the peer-to-peer network come to agreement on addition and verification of blockchain data without relying on a third party (middle-man)

Provenance

Blockchain participants know where the asset came from and how it’s ownership has changed over time

Immutability

No participant can tamper with a transaction after it has been recorded to the ledger. If a transaction is in error, a new transaction must be used to reverse the error, and both transactions are then visible.

UseCases of Blockchain

CryptoCurrencies

Bitcoin, Ethereum, Zcash and many many more are digital currencies built using blockchain technology. Cryptocurrency is not legal tender and it is not backed by a central government or bank, the values of these cryptocurrencies are agreed upon (based on buying and selling activity) at crypto exchanges (often unregulated) like coinbase and binance. Hence, it is advisable to be cautious while investing in these currencies.

Cross-Border Transactions

According to McKinsey research, it takes 3–5 business days on average to complete a cross-border transaction, with multiple steps involved along the way, each of whom charges 2–3% transaction fees. Using blockchain technology can make the transfer faster, cheaper and makes it possible to track payments while they are in transit.

Decentralized Applications

The ethereum blockchain allows for creation of truly decentralized applications, dApps. Unlike traditional apps where data is centralized in one location, dApps store transaction and other data directly on blockchain’s distributed ledger, which removes single points of failure and makes for more transparent operation.

Sensitive Data Storage

The decentralized nature of blockchain makes it possible to store sensitive data such as health records and other personal data and allow different stakeholders to access data only if they are authorized to. Sensitive data is encrypted and stored on blockchain and each access to this data can be audited by following the transaction trail.
