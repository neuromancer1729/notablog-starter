Getting started with Dino

Posted on Wed, Jun 24, 2020 tutorial web

Dino

Deno is a secure TypeScript runtime built on V8, the Google runtime engine for JavaScript.

It was built with:

  • Rust (Deno’s core was written in Rust, Node’s in C++)
  • Tokio (the event loop written in Rust)
  • TypeScript (Deno supports both JavaScript and TypeScript out of the box)
  • V8 (Google’s JavaScript runtime used in Chrome and Node, among others)

In short, Dino offers a secure, productive and compact scripting environment for the modern programmer.

Features

Compactness

Compactness is perhaps the least understood property of successful open source projects but it might be one of the most important.

Javascript got popular because of distribution monopoly (runtime available in all browsers) and a passing misnomerised resemblance to Java software whereas Python got popular because of compactness.

Compactness is what differentiates a meme from an image.

Compactness is the property that a design can fit inside a human being's head. Compact software tools have all the virtues of physical tools that fit well in the hand.

Dino's creator Ryan Dahl has embedded several design feature inside the framework to ensure this property

1.) Each Dino project to only ship a single executable

2.) Using modules references as URLs or paths instead of npm

import { fire, mystic } from "https://website.com/pokemons.ts";

Using ESMOdules syntax means you no longer have to require() , you can simply import x from y.

3.) Offline support by default - Remote code is fetched and cached on first execution, and never updated until the code is run with the --reload flag.

Security

Node by default has full access to your file system, where as Dino requires explicit permissions for file, network, and environment access.

Your machine is as secure as the least secure npm package in /usr/local

Full list of permission flags available on dino

> dino run -h

By default all programs are run in sandbox without access to disk, network or
ability to spawn subprocesses.
  deno run https://deno.land/std/examples/welcome.ts

Grant all permissions:
  deno run -A https://deno.land/std/http/file_server.ts

Grant permission to read from disk and listen to network:
  deno run --allow-read --allow-net https://deno.land/std/http/file_server.ts

Grant permission to read allow-listed files from disk:
  deno run --allow-read=/etc https://deno.land/std/http/file_server.ts

Deno allows specifying the filename '-' to read the file from stdin.
  curl https://deno.land/std/examples/welcome.ts | target/debug/deno run -

USAGE:
    deno run [OPTIONS] <SCRIPT_ARG>...

OPTIONS:
    -A, --allow-all                    Allow all permissions
        --allow-env                    Allow environment access
        --allow-hrtime                 Allow high resolution time measurement
        --allow-net=<allow-net>        Allow network access
        --allow-plugin                 Allow loading plugins
        --allow-read=<allow-read>      Allow file system read access
        --allow-run                    Allow running subprocesses
        --allow-write=<allow-write>    Allow file system write access
        --cached-only                  Require that remote dependencies are already cached
        --cert <FILE>                  Load certificate authority from PEM encoded file
    -c, --config <FILE>                Load tsconfig.json configuration file
    -h, --help                         Prints help information
        --importmap <FILE>             UNSTABLE: Load import map file
        --inspect=<HOST:PORT>          activate inspector on host:port (default: 127.0.0.1:9229)
        --inspect-brk=<HOST:PORT>      activate inspector on host:port and break at start of user script
        --lock <FILE>                  Check the specified lock file
        --lock-write                   Write lock file. Use with --lock.
    -L, --log-level <log-level>        Set log level [possible values: debug, info]
        --no-remote                    Do not resolve remote modules
    -q, --quiet                        Suppress diagnostic output
    -r, --reload=<CACHE_BLOCKLIST>     Reload source code cache (recompile TypeScript)
        --seed <NUMBER>                Seed Math.random()
        --unstable                     Enable unstable APIs
        --v8-flags=<v8-flags>          Set V8 command line options. For help: --v8-flags=--help

ARGS:
    <SCRIPT_ARG>...    script args

Unless specifically allowed, scripts can't access files, the environment, or the network. — deno.land

Productivity

In terms of dev-x Dino comes pre-packaged with a rich set of built-in Utilities

  • dependency inspector (deno info)
  • code formatter (deno fmt)
  • unit testing as well as code linting tooling

Browser compatibility: The subset of Deno programs which are written completely in JavaScript and do not use the global Deno namespace (or feature test for it), ought to also be able to be run in a modern web browser without change.
