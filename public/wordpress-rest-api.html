Wordpress rest API tutorial

tutorial

When you do a fresh install of wordpress it doesn't contain the .htaccess file. You can create it by editing the permalink structure from settings.

Now if you try to call the rest API you may get an error saying

ERROR: Wordpress Error: Sorry, you are not allowed to create posts as this user.
Error: Wordpress Error: Sorry, you are not allowed to create posts as this user.
    at Object.wordpressApiRequest (/usr/local/lib/node_modules/n8n/node_modules/n8n-nodes-base/dist/nodes/Wordpress/GenericFunctions.js:35:15)
    at processTicksAndRejections (internal/process/task_queues.js:97:5)

This is because by default only cookie based authentication is enabled in wordpress, hence a plugin is needed to enable username password based auth.

To solve this you need add this plugin

WP-API/Basic-Auth

This plugin adds Basic Authentication to a WordPress site. Note that this plugin requires sending your username and password with every request, and should only be used over SSL-secured connections or for local development and testing. Without SSL we strongly recommend using the OAuth 1.0a authentication handler in production environments.

https://github.com/WP-API/Basic-Auth

Download the entire repo as a zip file and upload it to your plugin directory

Activate the plugin and you're good to go!

Now you can proceed to interact with your wordpress instance via API

const WPAPI = require( 'wpapi/superagent' );

// fetching posts
var wp = new WPAPI({ endpoint: 'http://src.wordpress-develop.dev/wp-json' });
wp.posts().get()
    .then(function( data ) {
        // do something with the returned posts
    })
    .catch(function( err ) {
        // handle error
    });

//creating posts

// You must authenticate to be able to POST (create) a post
var wp = new WPAPI({
    endpoint: 'http://your-site.com/wp-json',
    // This assumes you are using basic auth, as described further below
    username: 'someusername',
    password: 'password'
});
wp.posts().create({
    // "title" and "content" are the only required properties
    title: 'Your Post Title',
    content: 'Your post content',
    // Post will be created as a draft by default if a specific "status"
    // is not specified
    status: 'publish'
}).then(function( response ) {
    // "response" will hold all properties of your newly-created post,
    // including the unique `id` the post was assigned on creation
    console.log( response.id );
})
